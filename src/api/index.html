HnS Login

How to use HnS Netai API:

// In order to use HnS Netai API you'll need to register you're website and request an API key. You'll then authenticate your website by calling the following function.

HNS.init({"apikey":"yourapikey"});

// You can manually call the login and logout functions by the HNS.login and HNS.logout functions.

$("#custom-login-link").click(function(){
	HNS.login();
});
$("#custom-logout-link").click(function(){
	HNS.logout();
});

// In order to include hns login and logout buttons you'll need to include a tag with an id of #hns-login-button or #hns-logout-button.

<div id="hns-login-button">HnS Login</div>
<div id="hns-logout-button">HnS Logout</div>

// You can also declare your button tag by using custom id's or classes as long as you pass them to the HNS.init function.

<div id="custom-login">HnS Login</div>
<div class="custom-logout">HnS Logout</div>

you pass these values to HNS.init like this (you don't have to include both the login and logout only the one's you want to use).

HNS.init({"apikey":"yourapikey","logintarget":"#custom-login","logouttarget":".custom-logout"});

// Application behavior can be set via HNS.init as well such as the logout action. If you set logoutaction to true the login screen will be displayed whenever the user is logged out as well as immediately after the user logs out.

HNS.init({"apikey":"yourapikey","logoutaction":true});

// After the user has logged in you will be provided with data for the currently logged in user and/or possibly the info you request. I haven't figured out how I will represent images. It could be full path name, full image name, or just the image name. this will be an array and the first element in the array will be the default image. All user images will be stored in the same directory. All user data will be cleared when they are logout.

HNS.user = {
	user_id: "1",
	username: "Andrew",
	firstname: "Andrew",
	middlename: "Anthony",
	lastname: "Gerst",
	fullname: "Andrew Anthony Gerst",
	images: "['default_image','image2','image3']"
};

// This will work fine on a website with any design but I recommend the following layout by splitting the design into both a logged in and logged out version especially if you are using html. This allows you to easily work with user accounts and displaying different view depending on logged in status. You handle the behavior using the HNS callback functions in the following manner. HNS API applys hnsin and hnsout classes to the body element that will let you style your website differently and specifically for logged in or logged out designs.

<style>
#out, #in { display: none }
</style>
<body>
<div id="out"></div>
<div id="in"></div>
</body>

// You can track and control your website based on the status of being logged in or logged out by using the functions HNS.loggedIn and HNS.loggedOut.

HNS.loggedIn = function(){
	$("#in").show();
	$("#out").hide();
	log("performing login");
	alert("Hello "+HNS.user.fullname+" welcome to our website!");
};
HNS.loggedOut = function(){
	$("#out").show();
	$("#in").hide();
	log("performing logout");
};
